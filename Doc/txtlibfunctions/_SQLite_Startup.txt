###User Defined Function###
_SQLite_Startup

###Description###
Loads SQLite.dll

###Syntax###
#include <SQLite.au3>
_SQLite_Startup ( [ $sDll_Filename = "sqlite3.dll" [, $bUTF8ErrorMsg = False ]] )


###Parameters###
@@ParamTable@@
$sDll_Filename
	[optional] Dll filename
$bUTF8ErrorMsg
	[optional] to force ConsoleWrite to display UTF8 chars
@@End@@

###ReturnValue###
@@ReturnTable@@
Success:	Returns path to SQLite3.dll
Failure:	@error = 1
@@End@@

###Remarks###
Requieres SQLite3.dll in @ScriptDir or @SystemDir.

If SQLite3.dll.au3 is included the dll will be created in @SystemDir.
If not successfull a temporary file will be created that will be destroyed at _SQLite_Shutdown().

If $bUTF8ErrorMsg is not zero prints UTF8 formated string prints to console.
This allows application as SCiTE to display those characters if configures with output.code.page=65001.
This is the case when running SCiTE under a non-english version as Vista(fr).
