###User Defined Function###
_WinAPI_RegQueryMultipleValues

###Description###
获取与打开的注册表键相关的值列表的数据和类型

###Syntax###
#Include <WinAPIEx.au3>
_WinAPI_RegQueryMultipleValues( $hKey, ByRef $aValent, ByRef $tData [, $iStart [, $iEnd]] )

###Parameters###
@@ParamTable@@
$hKey
   打开的注册表键的句柄. 该键必须以KEY_QUERY_VALUE访问权限打开. 该句柄须由_WinAPI_RegCreateKey()或_WinAPI_RegOpenKey()函数打开. 也可为下列预定义键之一:
   $HKEY_CLASSES_ROOT
   $HKEY_CURRENT_CONFIG
   $HKEY_CURRENT_USER
   $HKEY_LOCAL_MACHINE
   $HKEY_PERFORMANCE_DATA
   $HKEY_USERS
$aValent
   包含获取的值的名称的二维数组([valuename1, 0, 0, 0], [valuename2, 0, 0, 0], ... [valuenameN, 0, 0, 0]). 对于输入, 不使用1, 2和3数组元素, 但数组大小为[n][4], 否则, 函数失败. 另外, 如果指定键中不含任意指定值函数失败.
$tData
   接收数据的缓冲区. 如果函数成功, 缓冲区为一个接收每个值的数据的"byte[n]"结构. 在使用完函数返回的数据前不要改变此结构. 在那之后, 可释放由该结构分配的内存.
$iStart
   开始查询的数组索引
$iEnd
   结束查询的数组索引
@@End@@

###ReturnValue###
@@ReturnTable@@
成功: 复制到缓冲区的字节数量. $aValent数组包含下列数据:
    [n][0] - 值名(不改变).
    [n][1] - 数据的字节大小.
    [n][2] - 参数指向的缓冲区中的数据的指针.
    [n][3] - 数据类型($REG_*).
失败: 返回0并设置@error非0, @extended标记可包含非0系统错误代码.
@@End@@

###Remarks###
为了防止过多的系列化, 由该函数返回的数据总数不能超过一兆字节
