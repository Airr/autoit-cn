###User Defined Function###
_FTP_Command

###Description###
发送一个命令到 FTP 服务器.

###Syntax###
#Include <FTPEx.au3>
_FTP_Command($l_FTPSession, $s_FTPCommand [, $l_Flags = 0x00000001 [, $l_ExpectResponse = 0 [, $l_Context = 0]]])

###Parameters###
@@ParamTable@@
$l_FTPSession
	由 _FTP_Connect() 返回的回话名
$s_FTPCommand
	要发送到FTP服务器的命令字符串
$l_Flags
	[可选参数] $FTP_TRANSFER_TYPE_ASCII(文本传输) 或者 $FTP_TRANSFER_TYPE_BINARY(二进制传输)
$l_ExpectResponse
	[可选参数] Data socket for response in Async mode.参考备注
$l_Context
	[可选参数] A variable that contains the application-defined value that associates this search with any application data.
	This is only used if the application has already called _FTP_SetStatusCallback() to set up a status callback function.
@@End@@

###ReturnValue###
@@ReturnTable@@
成功:	返回一个标识(identifier).
失败:	返回 0 并设置 @ERROR.
@@End@@

###Remarks###
Command Examples: depends on server syntax/
下列命令:二进制传输, ASCII 传输, 被动传输模式 (用于防火墙) 'type I' 'type A'  'pasv'

If $l_ExpectResponse is set to TRUE, a pointer to a handle that is created if a valid data socket is opened is returned in @extended.

###Related###
_FTP_Connect, _FTP_SetStatusCallback
