###Function###
FileOpen

###Description###
打开指定的文件以供读写.

###Syntax###
FileOpen ( "文件名" [, 模式 ] )


###Parameters###
@@ParamTable@@
文件名 
	要打开的文件名.
模式
	[可选参数] 指定以何种模式(读或写)打开文件:
	可以是下列几种:
	  0 = 只读模式(默认)
	  1 = 写入模式(附加数据到文件尾部)
	  2 = 写入模式(先删除之前的内容)
	  8 = 如果目标目录不存在就创建(参考注意).
	  16 = 强制使用二进制(字节)模式(参考注意)
	  32 = 使用 Unicode UTF16 小编码读写模式,读取不会覆盖存在的BOM.
	  64 = 使用 Unicode UTF16 大编码读写模式,读取不会覆盖存在的BOM.
	  128 = 使用 Unicode UTF8 (带BOM)读写模式,读取不会覆盖存在的BOM.
	  256 = 使用 Unicode UTF8 (无BOM)读写模式.
	  16384 = 当打开一个文件读取时(文件没有BOM), 使用完整文件 UTF8 检测. 如果没有使用这一模式,则只会检测文件最前端的 UTF8 标志.
	文件夹路径必须存在(如果没有指定模式 '8' - 见注释).
@@End@@

###ReturnValue###
@@ReturnTable@@
成功:	返回一个文件"句柄",用于后来的文件函数调用.
失败:	返回值为 -1,说明有错误发生.
@@End@@


###Remarks###
<ul>
<li>文件句柄必须由 <a href="FileClose.htm">FileClose()</a> 函数关闭.</li>
<li>一个文件可能因为访问权限或者文件属性而失败.</li>
<li>默认状态下写入文本是 ANSI 编码 - 可以使用 unicode 标志修改它. 当使用 Windows 系统默认模式(在 AutoIt 中执行最少转换/最快写入的模式)写入 Unicode 文件的时, 编码为 UTF16 LE(Little Endian,小编码) (模式32)(官方另一说是UTF-8最快,参考更新日志).</li>
<li>以写入模式打开一个文件时,如果它不存在,文件将会被创建.默认情况下,如果目录结构不存在,文件将不会创建.可使用标志修改.<li>
<li>当同时读写一个相同的文件句柄, 必须使用 <a href="FileSetPos.htm">FileSetPos()</a> 函数更新当前使用的文件的当前文件坐标.</li>
<li>一个文件可以指定FileOpen使用二进制标志对它进行二进制(字节)方式进行读取 - 这种方式以字节数而不是字符数计算读取数量. 当一个读取数量过大时可能导致AutoIt因为内存分配失败而停止工作.</li>
<li>汉化PS:到底啥是BOM?BOM是byte order mark的缩写,中文化就是字节顺序标记.SB点说就是UNICODE文件的标识,让编辑器能识别文件的格式.你用一个16进制编辑器打开一个UNICODE文件,会会看到文件最前有FF FE/EF BB BF等这样特定的标志,这就是BOM.记住,不是BOOM~~.
</ul>
