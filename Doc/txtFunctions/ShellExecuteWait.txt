###Function###
ShellExecuteWait

###Description###
使用 ShellExecute API 执行一个外部程序并暂停脚本运行,直到外部程序正常结束.

###Syntax###
ShellExecuteWait ( "文件名" [, "参数" [, "工作目录" [, "verb" [, 显示标志]]]] )


###Parameters###
@@ParamTable@@
文件名
	需要运行的文件名 (.EXE,.txt, .lnk, 等等).
参数
	[可选] 程序需要的运行参数. 空白("")为不使用参数.
工作目录
	[可选] 程序的工作目录. 空白 ("") 使用当前工作目录
verb
	[可选] 使用 "verb" , 公共 verbs:
	  open = 打开指定文件. 文件可以是可执行文件,文档,或者一个文件夹
	  edit = 启动一个编辑器编辑指定文档. 如果 文件名不是一个文档文件,函数将失败
	  print = 打印指定的文档文件. 如果"文件名"不是一个文档文件,  函数将失败
	  properties = 显示文件或者文件夹的属性
	  <b>See remarks for more information on the default behavior when a verb is not specified.</b>

显示标志
	[可选] 可执行程序的显示标志:
	  @SW_HIDE = 隐藏窗口
	  @SW_MINIMIZE = 最小化窗口
	  @SW_MAXIMIZE = 最大化窗口
@@End@@


###ReturnValue###
@@ReturnTable@@
成功:	返回外部程序结束时的退出代码.
失败:	返回0并设置@error为非0值.
@@End@@


###Remarks###
指定的程序运行之后,脚本会被暂停.直到指定的程序终止运行.

When no verb is specified the default verb is used.  The default verb is the verb configured in the registry.  If no verb is set as default in the registry then the "open" verb is used.  If the "open" verb is not present then the first verb listed in the registry is used (except on Windows 2000).


###Related###
ShellExecute, Run, RunWait, RunAs, RunAsWait
