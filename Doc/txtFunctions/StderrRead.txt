###Function###
StderrRead

###Description###
从 STDERR 流读出一个先前运行的子进程.

###Syntax###
StderrRead ( 进程id[, 取数 = false[, 二进制 = false]] )


###Parameters###
@@ParamTable@@
进程id
	子进程ID, 由一个早先调用 Run 所返回.
取数
	如果设为 true 函数不移除从流读出的字符.
二进制
	如果设为 true 函数读取二进制数据而非文本.(默认为文本)..
@@End@@

###ReturnValue###
@@ReturnTable@@
成功: 	返回读取的数据. @extended 包含读取 bytes(二进制字节) 的数量. 
失败: 	设置 @error 非零， 如果 EOF 达到(范围), STDERR 不为进程或其他差错重新传入.
@@End@@


###Remarks###
StderrRead 从子进程控制台标准输出流读取, 通常是控制台应用程序对屏幕的写入. 在对子进程调用 Run 期间，你想要读取标准 I/O 参数包括的数值 $STDERR_CHILD (4) 这个函数应该适当地工作 (见 <a href="Run.htm">Run</a> 函数).
StderrRead 不阻塞，它将会立刻返回. 为了要获取所有数据，它一定在一个循环中被调用.
在流上取数不会把数据从缓冲器移开, 然而，它确实能返回得到的数据.
默认数据以文本的格式被返回. 如使用"二进制"选项,数据将会以二进制格式返回.
