###Function###
StdinWrite

###Description###
对先前运行的子进程写一些字符到 STDIN 流.

###Syntax###
StdinWrite ( 进程id[, data] )


###Parameters###
@@ParamTable@@
进程id
	子进程ID, 由一个早先调用 Run 所返回.
数据
	要输出的数据. 可以是文本或二进制数据.
@@End@@

###ReturnValue###
@@ReturnTable@@
成功: 	返回被写字符的数目. 
失败: 	设置 @error 非零 如果 STDIN 不为进程或其他的差错被重新传.
@@End@@


###Remarks###
StdinWrite 为一个子进程写入控制台标准输入流, 通常是根据控制台的应用程序读取用户的输入，比如从键盘. 在对子进程调用 Run 期间，你想要读取标准 I/O 参数包括的数值 $STDERR_CHILD (1) 这个函数应该适当地工作 (见 <a href="Run.htm">Run</a> 函数).

第二参数可选项是 StdinWrite 写入到流的字串. 如果函数没有调用第二参数, StdinWrite 将关闭流并且随后的写入无效.

stream(流) 是最初输入、最初输出的缓冲器，可以任意限制其大小; 当这函数被调用时(除非它正在被调用关闭流),对要写入给流的更多字符没有存储空间， StdinWrite 函数将会阻塞 (中止) 而且不返回，直到子进程关闭流或者读取流允许写入足够字符的过程完成. 这意谓 AutoIt 进程将会被停止, 而且将会没有热键处理, GUI 消息, 等等. 到子进程从 STDIN 流读取为止.
