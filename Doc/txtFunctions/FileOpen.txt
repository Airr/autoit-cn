###Function###
FileOpen

###Description###
打开指定的文件以供读写.

###Syntax###
FileOpen ( "文件名", 模式)


###Parameters###
@@ParamTable@@
文件名 
	要打开的文件名.
模式
	指定以何种模式(读或写)打开文件:
	可以是下列几种:
	  0 = 只读模式
	  1 = 写入模式(附加数据到文件尾部)
	  2 = 写入模式(先删除之前的内容)
	  4 = Raw 模式读取数据
	  8 = 如果目标目录不存在就创建(参考注意).
	  16 = 强制使用二进制(字节)模式读取与写入
	  32 = 使用 Unicode UTF16 小编码读写模式,读取不会覆盖存在的BOM.
	  64 = 使用 Unicode UTF16 大编码读写模式,读取不会覆盖存在的BOM.
	  128 =使用 Unicode UTF8 读写模式,读取不会覆盖存在的BOM.
	如果文件不存在,使用两个写入模式将会创建文件. 当然,文件夹路径必须存在(如果没有指定模式 '8' - 见注释).
@@End@@

###ReturnValue###
@@ReturnTable@@
成功:	返回一个文件"句柄",用于后来的文件函数调用.
失败:	返回值为 -1,说明有错误发生.
@@End@@


###Remarks###
一个文件只能是读或写模式; 不能同时存在读,写两种模式.
一些文件属性可能导致不能打开文件.
以写入模式打开一个文件时,如果它不存在,文件将会被创建.
当一个文件操作完成,请调用 FileClose 函数关闭文件. 虽然 Autoit 正常情况下会在程序终止时关闭所有的文件,但是明确的调用 FileClose 是一个好主意.

默认状态下写入文本是 ANSI 编码 - 可以使用 unicode 标志修改它. 当使用 Windows 系统默认模式(在 AutoIt 中执行最少转换/最快写入的模式)写入 Unicode 文件的时, 编码为 UTF16 LE(Little Endian,小编码) (模式32)..

当使用模式=4 (Raw 模式读取数据)时,如果文件名定义为 "\\.\A:" 将会读取一个软盘扇区.读取的字符数必须为扇区大小的倍数(512).

不存在的目录结构:
------------------------------------
默认情况下,如果目录结构不存在,文件将不会创建.
如果要覆盖数据,使用模式 '1' 或 '2' 并结合模式 '8'!
例如模式标志 '9' (1 + 8) ,检查目录结构,如果它不存在就自动创建.然后使用附加模式打开文件.
