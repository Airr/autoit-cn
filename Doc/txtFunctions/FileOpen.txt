###Function###
FileOpen

###Description###
打开指定的文件以供读写.

###Syntax###
FileOpen ( "文件名", 模式)


###Parameters###
@@ParamTable@@
文件名 
	要打开的文件名.
模式
	指定以何种模式(读或写)打开文件:
	可以是下列几种:
	  0 = 只读模式
	  1 = 写入模式(附加数据到文件尾部)
	  2 = 写入模式(先删除之前的内容)
	  4 = Raw 模式读取数据
	  8 = 如果目标目录不存在就创建(参考注意).
	  16 = 强制使用二进制(字节)模式读取与写入
	  32 = 使用 Unicode UTF16 小编码读写模式,读取不会覆盖存在的BOM.
	  64 = 使用 Unicode UTF16 大编码读写模式,读取不会覆盖存在的BOM.
	  128 =使用 Unicode UTF8 读写模式,读取不会覆盖存在的BOM.
	如果文件不存在，使用两个写模式将会产生文件.文件夹路径必须存在 (除模式 '8' 外- 见注释).
@@End@@

###ReturnValue###
@@ReturnTable@@
成功:	返回一个文件"句柄",用于后来的文件函数调用.
失败:	返回值为 -1,说明有错误发生.
@@End@@


###Remarks###
一个文件只能是读或写模态; 不能同时存在读、写两种模态.
一些文件属性可能不会打开.
以写入模式打开一个文件时,如果它不存在，文件将会被建立.
当与一个文件合作完成,调用 FileClose 函数关闭文件. 虽然 Autoit 通常在终端关闭所有的文件,但是调用 FileClose 是一个好主意.

默认模态当写入文本是 ANSI 时 - 使用 unicode 标志改变. 当在 Windows 系统默认模态(在 AutoIt 由于最少的和最快速的转换)写入 unicode 文件的时候， 是 UTF16 Little Endian (mode 32)..

当使用模态=4 (Raw 模式读取数据)时，文件名定义为 "\\.\A:" 读取一个软盘扇区数一定是扇区大小的倍数(512).

非现有目录结构时:
------------------------------------
如果预先设定的目录结构不存在，文件将不被建.
要重写，使用模态 '1' 或 '2'加上模态 '8'!
例如模态标志 '9' (1 + 8) 检查目录结构、如果它不存在就自动创建、然后打开启附加文件.
